# 기술 요구 사항 문서 (TRD)

## 1. 실행 기술 요약
- **프로젝트 개요**: 컨퍼런스 참석자와의 커뮤니케이션을 간소화하기 위한 웹 기반 자동화 도구를 개발합니다. 주요 기술적 접근 방식은 React를 사용하여 사용자 인터페이스를 구축하고, Python Flask 백엔드를 통해 데이터 처리 및 API를 제공하며, MySQL 데이터베이스에 데이터를 저장하고, Firebase Authentication으로 사용자 인증을 관리합니다.
- **핵심 기술 스택**: React, Python, Flask, MySQL, Firebase Authentication, Flask-Uploads
- **주요 기술 목표**:
    - 성능: 빠른 응답 시간과 효율적인 데이터 처리 보장
    - 확장성: 많은 수의 참석자 데이터와 트래픽을 처리할 수 있도록 설계
    - 안정성: 안정적인 시스템 운영 및 데이터 무결성 유지
- **중요 기술 가정**:
    - Google Sheets API 및 Outlook SMTP 서버와의 안정적인 연결을 가정합니다.
    - Firebase Authentication의 안정적인 인증 서비스를 가정합니다.
    - 사용자들은 표준 웹 브라우저를 사용하여 시스템에 접속한다고 가정합니다.

## 3. 시스템 아키텍처 설계

### 최상위 빌딩 블록
- **프론트엔드 (React)**:
    - 사용자 인터페이스 및 사용자 상호 작용 처리
    - API 요청을 백엔드로 전송
    - 데이터 시각화 (대시보드)
- **백엔드 (Python Flask)**:
    - API 엔드포인트 제공
    - Google Sheets 데이터 처리 및 관리
    - 이메일 스크립트 처리 및 전송
    - 데이터베이스 상호 작용
    - Firebase Authentication을 통한 사용자 인증
- **데이터베이스 (MySQL)**:
    - 참석자 데이터, 이메일 스크립트, 전송 결과 저장
- **Firebase Authentication**:
    - 사용자 인증 및 권한 관리

### 최상위 컴포넌트 상호 작용 다이어그램

```mermaid
graph TD

```

- 프론트엔드 (React)는 사용자의 요청에 따라 백엔드 (Flask)에 API 요청을 전송합니다.
- 백엔드 (Flask)는 데이터베이스 (MySQL)에 데이터를 저장하거나 조회합니다.
- 백엔드 (Flask)는 Firebase Authentication을 사용하여 사용자를 인증하고 권한을 확인합니다.
- 백엔드 (Flask)는 Google Sheets API를 사용하여 Google Sheets에서 참석자 데이터를 가져옵니다.
- 백엔드 (Flask)는 Outlook SMTP Server를 사용하여 이메일을 전송합니다.

### 데이터 흐름 및 통신 패턴
- **클라이언트-서버 통신**: React 프론트엔드는 Flask 백엔드에 API 요청을 보내고, 백엔드는 JSON 형식으로 응답합니다.
- **데이터베이스 상호 작용**: Flask 백엔드는 SQLAlchemy ORM을 사용하여 MySQL 데이터베이스와 상호 작용합니다.
- **외부 서비스 통합**: Google Sheets API 및 Outlook SMTP 서버는 Python 라이브러리를 통해 통합됩니다.
- **실시간 통신**: 대시보드 업데이트를 위해 필요에 따라 WebSocket 또는 SSE를 구현할 수 있습니다.
- **데이터 동기화**: 데이터베이스와 Google Sheets 간의 데이터 동기화를 위해 주기적인 데이터 업데이트 작업을 수행합니다.
